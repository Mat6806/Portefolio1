SAE 1.04 – Se présenter sur internet Semestre 1

HTML / CSS Identité numérique
›

Présentation

Dans cette SAE, nous devions réaliser une page web non dynamique présentant notre profil professionnel ainsi qu’une page sur notre projet professionnel.

Les consignes incluaient une réflexion sur l’identité numérique, la trace laissée sur Internet et la construction d’un descriptif personnel (avec une partie en anglais sur un centre d’intérêt).

Réalisation

Pour mener à bien la SAE, je me suis concentré sur la partie technique :

  • écriture du code HTML et CSS ;
  • mise en forme d’une structure de navigation simple entre les pages ;
  • choix d’un style visuel sobre avec des boutons sur fond bleu et des images de l’IUT en arrière‑plan sur chaque page.

La mise en page a été priorisée pour assurer un rendu propre avant de finaliser les contenus textuels.

Compétences visées

  • CE3.03 – Utiliser les outils numériques à bon escient
  • CE3.04 – Choisir les outils de développement adaptés

SAE 1.05 – Traiter des données Semestre 1

Python / Scapy Réseaux & sécurité
›

Présentation

Cette SAE consistait à réaliser deux scripts avec la bibliothèque Scapy en Python pour manipuler des paquets réseau :

  • un programme de découverte des hôtes sur un réseau ;
  • un programme permettant une attaque de l’homme du milieu (MITM).

L’objectif était de consolider les connaissances réseau tout en développant les compétences de programmation réseau.

Réalisation

En binôme, nous avons réparti les tâches :

  • je me suis occupé du script man in the middle ;
  • mon binôme Simon a pris en charge le script de reconnaissance passive ;
  • les scripts ont été développés et testés sur une machine virtuelle Debian 12.

SAE 2.03 – Mettre en place une solution informatique pour l’entreprise Semestre 2

Django / Apache2 Virtualisation & base de données
›

Présentation

La SAÉ 2.03 avait pour objectif de réaliser et de déployer une solution d’accès à des données via un site web permettant la consultation, l’insertion et la manipulation de données stockées dans une base de données.

SAE 3.02 – Développer des applications communicantes Semestre 3

Python / PyQt6 Client / serveur multi‑langages
›

Présentation

Pour cette SAE, nous devions développer un serveur capable d’exécuter des programmes (C, C++, Java, Python) envoyés par un client. Le serveur devait recevoir le programme, le compiler si nécessaire, l’exécuter puis renvoyer la sortie au client.

Le client disposait d’une interface graphique pour envoyer les programmes et afficher les résultats.

Réalisation

J’ai choisi d’utiliser la bibliothèque PyQt6 pour la partie interface graphique du client. Les étapes principales ont été :

  • mise en place de la logique du client (envoi des programmes, réception des sorties) ;
  • intégration de l’interface graphique PyQt6 ;
  • développement du serveur chargé de gérer la compilation, l’exécution et le retour des résultats.

Compétences visées

  • CE3.01 – Programmer en étant à l’écoute des besoins du client
  • CE3.02 – Programmer en documentant le travail réalisé
  • CE3.04 – Choisir les outils de développement adaptés

Apprentissages critiques

  • AC23.03 – Utiliser un protocole réseau pour programmer une application client/serveur
  • AC23.05 – Accéder à un ensemble de données depuis une application et/ou un site web
